Docs

This section describes the API documentation for working with the payment service.

Token

The token is a unique key that is used to authenticate the user. To get started, go to the "Tokens" section and create a token key

Payments

The payment is a transaction that is made by the clients. This section describes the documentation for Payments methods.

Payment State

The payment state is a status that indicates the current state of the payment. The payment state can be one of the following values:

  • PENDING - The payment has been created and waiting for payment.
  • PAID - The payment has been paid successfully.
  • REJECTED - The payment has been rejected.
Payment Method

The payment method is a method that is used to make a payment. The payment method can be one of the following values:

  • CARD - The payment is made using a card.
  • QRCODE - The payment is made using a QR code.
Create Payment

To create a payment, you need to send a POST request to the endpoint /api/payments. The request body should contain the following fields: 

{
    "amount": 100.00,
    "method": "card",
    "token_key": "..."
}
Get Payment

To get a payment, you need to send a GET request to the endpoint /api/payments/{id}. The request should contain the payment id in the URL.
